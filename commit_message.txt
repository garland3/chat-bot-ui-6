Fix: Prevent API keys exposure in LLM configuration

Prevented API keys from being exposed via the `/api/llm_configs` endpoint.

Changes made:
- Introduced `LLMConfigPublic` Pydantic model in `app/services/llm_config_manager.py` to exclude `api_key`.
- Modified `LLMConfigManager.get_all_llm_configs` to return `LLMConfigPublic` objects.
- Updated `app/routers/llm_configs.py` to use `LLMConfigPublic` as the response model for `/api/llm_configs`.
- Added `from typing import List` in `app/routers/llm_configs.py` to resolve `NameError`.
